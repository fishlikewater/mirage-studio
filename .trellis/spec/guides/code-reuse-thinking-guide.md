# 代码复用思考指引

## 什么时候必须看

- 你准备新增常量、错误码、工具函数、查询模板、DTO、校验逻辑
- 你刚在多个文件做了类似修改
- 你觉得“这个应该以后还会再用”
- 你准备新增节点类型、工具定义、provider/model 能力、i18n key 或样式 token

## 检查问题

- 项目里是否已经有同类实现？
- 这个能力应该放在公共层、领域层还是模块内？
- 现在抽象是否真的减少重复，还是提前设计？
- 如果以后只会复用一次，是否值得抽取？
- 是否已有 `nodeControlStyles.ts`、`nodeToolbarConfig.ts`、`nodeRegistry.ts` 或模型 registry 可以复用？
- 是否应该更新现有 registry/codec，而不是新增一套平行配置？

## 最小动作

```bash
rg "相似关键字"
```

## 原则

- 先搜索，后新增
- 先复用，后抽象
- 先最小可用，后稳定沉淀
- 不为单次需求新建与现有 registry 平行的白名单或映射表
