# 编码前检查表

## 在动手前先确认

- 这次改动属于 `L0`、`L1` 还是 `L2`？
- 是否涉及 Tauri 命令、SQLite、AI provider、图片字段、异步链路或画布状态流转？
- 是否需要先补 OpenSpec？
- 是否已经明确成功标准和验证方式？
- 是否已经搜索过相似实现与可复用能力？
- 是否需要同步 `zh.json` / `en.json`？
- 是否需要同步 `.trellis/spec/`、OpenSpec 或发布说明？

## 如果涉及行为变化

- 是否已有 `proposal/spec/design/plan`？
- 是否已经明确输入、输出、错误、边界场景？
- 是否知道谁负责提交业务代码？
- 是否已列出需要验证的主路径与异常路径？
