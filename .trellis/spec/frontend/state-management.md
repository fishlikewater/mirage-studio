# 前端状态管理规范

## 目标

- 区分画布交互状态、项目持久化状态、模型/工具配置状态和组件局部状态
- 保持 Store、应用服务、Tauri 命令和持久化边界清晰

## 推荐做法

- 项目状态以 `projectStore` 为持久化入口，画布节点/边以 `canvasStore` 为主要交互入口。
- Store 只承载状态编排和轻量动作；工具执行、模型请求、图片处理等重业务逻辑放到 `application` 或专用服务。
- 画布节点类型、默认数据、菜单展示、连线能力以 `nodeRegistry.ts` 为单一真相源。
- 服务端/Tauri 状态优先交给项目统一命令封装处理
- 客户端临时状态尽量就近管理
- 全局状态只保留真正跨模块共享的状态
- 视口持久化走独立轻量队列，不回退到整项目 upsert。

## 检查项

- 是否把局部状态错误提升成全局状态
- 是否把服务端状态与本地状态混在一起
- 是否绕过 Store 或应用层直接偷改节点数据
- 是否在拖拽、缩放、输入时触发不必要的整项目保存
- 是否新增图片字段但没有同步 imagePool 编码/解码映射
