# 前端性能规范

## 核心原则

- 画布交互优先“流畅”，不要追求每一帧实时全量持久化。
- 拖拽、缩放、输入等高频路径禁止执行重持久化、重计算或大对象序列化。

## 画布与持久化

- 节点拖拽中不要写盘；拖拽结束再保存。
- 项目整量持久化使用防抖和 idle 调度队列，避免与交互争用主线程。
- viewport 持久化走独立轻量队列与独立命令 `update_project_viewport_record`。
- 视口更新要做归一化与阈值比较，过滤微小抖动写入。

## 图片数据

- 大图片场景避免重复 `dataURL` 转换。
- 节点渲染优先使用 `previewImageUrl`。
- 模型和工具处理使用原图 `imageUrl`。
- 新增图片字段必须同步 imagePool 编码/解码，避免重复持久化大字符串。

## React 状态与渲染

- Zustand selector 只选择必要字段，避免组件订阅整个大对象。
- 对明显昂贵的派生计算使用合适的缓存或拆分，但不要为普通值滥用 memo。
- 避免把大对象直接塞进依赖数组导致抖动。
- 快捷键、订阅、计时器等副作用必须正确清理。
