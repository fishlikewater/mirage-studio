# 后端异常处理规范

## 目标

- 调用方能稳定感知错误
- 内部错误与外部错误边界清晰

## 推荐做法

- Tauri 命令对外返回稳定错误结构或稳定错误文案，不把 Rust 内部错误原样泄露给 UI。
- 参数校验错误、业务错误、系统错误分层处理
- 对外返回稳定结构，对内保留足够上下文
- 不把底层异常原样泄露给调用方
- AI/provider 错误必须保留供应商、模型、请求阶段等排障上下文，但不得泄露 API Key。
- 持久化错误应区分“数据库不可用”“数据格式不可解码”“写入失败”等场景，方便 UI 给出可读提示。

## 典型检查项

- 是否区分“可预期业务失败”和“系统异常”
- 是否保留 traceId / requestId / correlationId
- 是否有统一异常兜底
- 是否记录了必要但不过量的日志
- 前端 `src/commands/**` 是否同步处理新增错误分支
