# 后端日志规范

## 目标

- 便于排查
- 便于追踪链路
- 不泄露敏感信息

## 推荐做法

- Rust 后端优先使用项目已有日志方式，记录命令名、provider/model、项目 id、节点 id 等排障上下文。
- AI 请求链路记录阶段与摘要信息，不记录完整提示词、图片 dataURL、API Key 或用户私密内容。
- 错误日志记录上下文，不重复打印同一异常
- 对账号、令牌、密钥、身份证号、手机号等敏感字段脱敏

## 不建议

- 直接输出完整请求体或完整响应体
- 使用临时 `print` / `console` 式调试输出长期留在主干
- 同一错误在多层重复打印
- 在前端长期保留用于调试节点数据的大对象日志
