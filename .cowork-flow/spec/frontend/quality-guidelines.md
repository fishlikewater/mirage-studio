# 前端质量规范

## 完成定义

至少满足：

- 构建、类型检查或等价静态验证通过。
- 关键交互已通过自动化测试、浏览器验证或明确的手工验证记录覆盖。
- 加载、空态、错误态、禁用态和权限不足状态已处理。
- 新增视觉、交互、接口或状态约定已同步到相关文档。

## 默认门禁

- 优先运行仓库已有的 lint、format、type-check、test 和 build 命令。
- 没有统一命令时，运行最接近变更面的最小可靠验证，并说明覆盖范围。
- 涉及布局、交互、响应式或可访问性时，使用真实浏览器或截图验证关键视口。
- 涉及共享类型、接口响应或权限状态时，同步验证前后端契约。

## 常见禁止模式

- 用 `any`、强制断言或空对象默认值掩盖真实类型问题。
- 组件未处理加载、空态、错误态、禁用态或权限不足。
- 交互依赖隐式副作用，缺少明确的数据流和清理逻辑。
- 把局部状态提升为全局状态，或把服务端状态复制成多份本地状态。
- 只验证理想路径，不验证失败、空数据和重复操作。
