# 前端开发规范

## 默认基线

- 前端代码以清晰的信息结构、稳定的交互状态和可维护的组件边界为核心。
- 契约来源优先级：后端接口、共享类型、设计系统、现有页面模式，然后才是本目录通用规则。
- 新增页面或组件应贴合现有路由、状态管理、样式系统和可访问性约定。
- 如果仓库没有前端代码，本目录规则仅作为未来引入前端时的默认约束。

## 使用原则

- 页面入口负责组织流程，业务组件负责局部交互，通用组件保持无业务假设。
- 服务端状态、客户端临时状态、权限状态和表单草稿应保持边界清晰。
- 所有用户可见状态都应考虑加载、空态、错误态、禁用态和权限不足。
- 变更接口或共享类型时，同步更新调用方、校验逻辑和展示边界。

---

## 默认完成标准

- 关键交互路径已验证。
- 组件文本、布局和状态在常见屏幕宽度下不重叠、不截断关键内容。
- 类型检查、构建或等价验证覆盖了本次改动。
- 没有用强制断言、全局状态或隐式副作用掩盖真实契约问题。
