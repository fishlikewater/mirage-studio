# 后端数据库规范

## 目标

- 数据模型、迁移、查询方式保持一致。
- 数据变更可审计、可回滚、可复现。
- 事务边界与业务动作一致，不用扩大事务掩盖建模问题。

## 默认规则

- 表结构变更必须通过迁移文件、版本化脚本或仓库既有机制管理。
- 查询优先使用仓库统一的数据访问方式；没有统一封装时，保持查询入口集中且可测试。
- 写操作应明确幂等性、并发影响、失败补偿和重试边界。
- 审计字段、逻辑删除、版本号、租户隔离等约定必须在同一数据域内一致。
- 原始 SQL、批量更新、跨表聚合和长事务都需要说明索引、锁、性能和回滚影响。

## 检查项

- 新表或新集合是否补齐主键、唯一约束、索引和必要审计信息。
- 复杂查询是否避免 N+1、隐式全表扫描和不受控分页。
- 事务中是否包含外部调用、慢查询或不必要的长时间锁持有。
- 数据迁移是否考虑历史数据、重复执行、失败恢复和线上回滚。
- 错误处理是否区分数据不存在、数据冲突、约束失败和系统异常。
