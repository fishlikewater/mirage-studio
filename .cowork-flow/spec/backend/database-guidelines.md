# 后端数据库规范

## 目标

- 模型、迁移、查询方式一致
- 数据变更可审计、可回滚、可复现

## 基本要求

- 表结构变更必须通过迁移文件管理
- 查询优先使用项目统一的数据访问方式
- 事务边界由业务动作定义，不要为“方便”扩大事务
- 审计字段、逻辑删除、版本号等约定要全项目一致

## 建议检查项

- 新表是否补齐主键、索引、审计字段
- 复杂查询是否可通过统一 API 或查询构造器表达
- 是否存在拼接 SQL / 条件遗漏 / N+1 查询
- 是否在错误层做了数据一致性补偿
