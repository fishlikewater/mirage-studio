# 后端质量规范

## 完成定义

至少满足：

- 编译、导入或构建检查通过。
- 关键业务路径有自动化测试或明确的手工验证记录。
- 新增或变更的错误路径、权限路径、数据边界已验证。
- 新增约定、契约或迁移影响已沉淀到 spec、任务记录或相关文档。

## 默认门禁

- 优先运行仓库已有的 lint、type-check、unit test、integration test 和 build 命令。
- 没有统一命令时，运行最接近变更面的最小可靠验证，并说明覆盖范围。
- 共享逻辑、公共 API、数据迁移、权限判断和异步任务需要比局部 UI 或文案改动更强的验证。
- 失败验证不得通过缩小断言、删除测试或跳过关键路径来规避。

## 常见禁止模式

- 未验证就宣称完成。
- 用硬编码绕过配置、权限或契约。
- 用临时兼容逻辑代替真实修复。
- 变更接口或数据结构却不更新文档和调用方。
- 捕获异常后静默吞掉错误或返回成功。
